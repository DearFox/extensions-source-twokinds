# TwoKinds extensions

Этот форк содержит исходный код нацеленный на добавления и улучшение поддержки комикса TwoKinds (два вида) для tachiyomi/mihon.

Цели форка:
- [x] Добавить поддержку twokinds.ru
- [x] Добавить нормальное разделение на главы для русской версии
- [x] Добавить нормальное разделение на главы для английской версии
- [x] Объеденить Английскую и Русскую версии расширения в одну мультиязычную с использованием `SourceFactory`
- [ ] Добавить поддержку Чешского перевода https://twokinds.cz/archive (отличающийся парсинг)
- [ ] Добавить поддержку Немецкого перевода https://www.twokinds.de/Archiv (отличающийся парсинг)
- [ ] Добавить поддержку Шведского перевода https://twokinds.se/?a=0&p=0 (отличающийся парсинг, возможно заброшенный перевод)
- [ ] добавить Португальский перевод https://twokindsbr.blogspot.com/p/arquivo.html (отличающийся парсинг, сложный парсинг архива)
- [ ] Добавить Украинский перевод (увы, но сайт не работает а в веб архиве не полный перечень страниц:  https://web.archive.org/web/20250406104948/https://twokinds.pp.ua/archive/)
- [ ] (?) Добавить фильтрацию филлеров как функцию
- [ ] (?) Добавить поддержку twokinds.xyz (не имеет много смысла, но почему бы и нет)
- [ ] (?) Другие языковые версии в будущем...

## Usage
**If you are new to repository/extensions, please read the [Keiyoushi Getting Started guide](https://keiyoushi.github.io/docs/guides/getting-started#adding-the-extension-repo) first.**

* You can add our repo by visiting the [Keiyoushi Website](https://keiyoushi.github.io/add-repo)
* Otherwise, copy & paste the following URL: https://raw.githubusercontent.com/keiyoushi/extensions/repo/index.min.json

## Requests

To request a new source or bug fix, [create an issue](https://github.com/keiyoushi/extensions-source/issues/new/choose).

Please note that creating an issue does not mean that the source will be added or fixed in a timely
fashion, because the work is volunteer-based. Some sources may also be impossible to do or prohibitively
difficult to maintain.

If you would like to see a request fulfilled and have the necessary skills to do so, consider contributing!
Issues are up-for-grabs for any developer if there is no assigned user already.
